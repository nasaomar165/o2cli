shield SECURITY HARDEDNING — v0.2.0

Every command passes through 5 validation layers

Defense-in-depth: if one layer fails, the next catches it. All mitigations tagged with traceable IDs.

1

Parse & Validate

H-01 M-04

Strip markdown and prompt markers. Enforce MAX_CMD_LENGTH (4096). Check against _SAFE_COMMANDS positive allowlist. Detect shell metacharacters.

2

Allowlist Check

M-08

Match against user-defined patterns using anchored regex. Reject overly permissive patterns like *. Commands matching allowlist but containing metacharacters are still flagged.

3

Static Danger Detection

30+ patterns
Regex-based detection across 10 categories:
rm -rf / fork bombs mkfs / dd shutdown curl | sh sudo chmod 777 history -c command injection
4

LLM Safety Check (optional)

--llm-safety

Ask the LLM to assess risk: LOW / MEDIUM / HIGH. HIGH risk commands are automatically blocked.

5

User Confirmation

H-04

Action menu: Execute / Copy / Retry / Skip. Trusted directories may skip confirmation.

warning

SSRF Protection H-02

URL validation blocks private/link-local IP ranges (10.x, 172.16.x, 192.168.x, 169.254.x). Internal domains (.internal, .local, metadata.google.internal) are blocked. localhost / 127.0.0.1 are explicitly allowed for local LLM backends.

Commands Reference

Eight core commands. Zero bloat.

Convert natural language to a terminal command

The core command. Describe what you want — O2Cli generates the exact shell command with an AI explanation. Dry run by default for safety.

--execExecute immediately
--dry-runShow only, never execute
--copyCopy to clipboard
--outputRaw text for piping
--llm-safetyLLM risk assessment
--contextgit, ls, env, or text
EXAMPLES 
# Simple query
o2cli ask "find all .log files over 100MB"

# Execute with git context
o2cli ask "show recent commits" --exec --context git

# Use a different model
o2cli ask "list docker containers" -m llama3.2

# Template variable
o2cli ask "compress {file}" --exec

# Pipe output
o2cli ask "find my IP address" --output

View or change configuration

Manage backend settings, shell preferences, security policies, profiles, and config bundles.

--showDisplay current config
--wizardInteractive setup
--allowAdd allowlist pattern
--trust-dirMark directory as trusted
--export/--importConfig bundle I/O
EXAMPLES 
# View config
o2cli config --show

# Set provider
o2cli config -p ollama -m llama3.2

# Add allowlist pattern
o2cli config --allow "git *"
o2cli config --allow "docker ps"

# Trust a directory
o2cli config --trust-dir /home/user/projects

# Export/Import config bundle
o2cli config --export > backup.json
o2cli config --import-file backup.json

Searchable command history

View, search, star, re-run, and export your command history. All stored with 0600 permissions.

--searchFilter by keyword
--starToggle star on entry
--rerunRe-run a past command
--statsUsage statistics
--exportExport as JSON or CSV

o2cli history --export json

Task aliases with template support

Save frequently-used task descriptions as named shortcuts. Template variables are filled at runtime with validated input.

setSave a new alias
runExecute a saved alias
listList all aliases
removeDelete an alias
EXAMPLES 
# Set alias
o2cli alias set ports "list all open ports"

# With template variable
o2cli alias set kill-port "find and kill process on {port}"

# Run alias
o2cli alias run ports

# List & remove
o2cli alias list
o2cli alias remove ports

Safe Undo

When a command succeeds, O2Cli suggests an undo that moves files to .o2cli/trash/ instead of rm -rf.

TEMPLATE VARIABLES
$ o2cli ask "compress {file}" --exec
Prompt: file = mydata.tar
Result: tar -czf mydata.tar.gz mydata.tar
$ o2cli ask "find process on {port}" --exec
Prompt: port = 8080
Result: lsof -i :8080
SAFE UNDO
ORIGINAL
mv src dest
mkdir mydir
touch newfile
cp src dest
UNDO
mv dest src
mv mydir .o2cli/trash/
mv newfile .o2cli/trash/
mv dest .o2cli/trash/

Quick Start

Up and running in under a minute. Dependencies auto-install on first run.

o2cli — quickstart
# 1. Clone and enter the repo
git clone https://github.com/nasaomar165/o2cli && cd o2cli

# 2. Make executable (optional)
chmod +x o2cli.py

# 3. First run — interactive setup wizard
python o2cli.py

# 4. Generate a command (dry run by default)
python o2cli.py ask "list all python files in this directory"

# 5. Generate and execute
python o2cli.py ask "find process on port 8080" --exec

# 6. Interactive chat
python o2cli.py chat

Full Hardening Inventory

Every mitigation tagged with a traceable identifier for audit.

C-01
shell=True eliminated

All subprocess calls use explicit argument lists with shell=False.

C-02
Template variable quoting

Values validated against strict allowlist and shlex.quote()'d before substitution.

C-03
Multi-step validation

Multi-step commands validated per-step with length and count limits.

H-01
Positive-allowlist parser

Replaces _clean_cmd() with _SAFE_COMMANDS set.

H-02
SSRF protection

Blocks private/link-local IP ranges.

H-03
Env var validation

Overrides validated and logged at WARN level.

H-04
/run confirmation

Chat /run requires explicit user confirmation.

H-05
Config bundle validation

Validates against JSON Schema before writing.

H-06
Safe context collection

_collect_context() uses shell=False with validated paths.

M-01
Secure file permissions

All files created with 0600 (owner-only) permissions.

M-02
Safe undo

Moves files to .o2cli/trash/ instead of rm -rf.

M-04
Hard limits

MAX_CMD_LENGTH (4096) and MAX_STEPS (20) enforced.

M-06
Debug log redaction

Debug logging redacts API keys and truncates payloads.

M-07
History privacy

0600 permissions; --no-history mode added.

M-08
Anchored regex allowlist

Anchored regex patterns reject overly permissive rules.

L-01
Welcome file bug fix

Removed erroneous _WELCOME_SHOWN_FILE.unlink() on exit.

L-02
LLM call budget

Per-session budget (default: 100) prevents runaway usage.

L-03
In-process completion

Completion command uses in-process handler, no re-invocation.
